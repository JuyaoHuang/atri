## 快速开始

### 创建新角色的 3 个步骤

```bash
# 1. 创建角色定义文件
cd D:\Coding\GitHub_Resuorse\emotion-robot\atri
touch prompts/persona/my_character.md

# 2. 编写角色设定（见下文格式说明）
# 使用文本编辑器编辑 prompts/persona/my_character.md

# 3. 准备头像文件（可选）
# 将头像图片放到前端项目：atri-webui/public/avatars/my_character.png
```

**就这么简单！** 后端会自动扫描 `prompts/persona/` 目录，前端调用 `GET /api/characters` 即可获取新角色。

## 角色文件格式

### 基本结构

角色文件使用 **Markdown + YAML Frontmatter** 格式：

```markdown
---
name: 角色显示名称
avatar: avatar_filename.png
greeting: 首次问候语
description: 角色简介
---

# 角色设定

这里是角色的系统提示词（System Prompt）...
```

### 格式规范

| 组成部分 | 说明 | 必填 |
|---------|------|------|
| **Frontmatter** | YAML 格式的元数据，用 `---` 包裹 | ❌ 可选 |
| **Body** | Markdown 格式的系统提示词 | ✅ 必填 |

**重要**：
- Frontmatter 必须在文件开头
- 必须用 `---` 包裹（开头和结尾各一行）
- Body 部分会原样作为 LLM 的 System Prompt

---

## 字段说明

### Frontmatter 字段

#### 1. `name` - 角色显示名称

**类型**：`string`  
**必填**：❌ 否（默认使用文件名）  
**用途**：前端显示的角色名称

**示例**：
```yaml
name: 亚托莉
```

**前端使用场景**：
- 角色选择器列表
- 聊天界面顶部
- 消息发送者名称

**默认值**：如果不提供，使用 `character_id`（文件名，不含 `.md`）

---

#### 2. `avatar` - 头像文件名

**类型**：`string | null`  
**必填**：❌ 否（默认 `null`）  
**用途**：角色头像图片的文件名

**示例**：
```yaml
avatar: atri.png
```

**前端使用场景**：
- 角色卡片头像
- 聊天消息气泡旁的头像
- 角色选择器缩略图

**文件位置**：
- **推荐**：`atri-webui/public/avatars/atri.png`
- **访问方式**：`/avatars/atri.png`

**支持格式**：
- ✅ PNG（推荐，支持透明背景）
- ✅ JPG/JPEG
- ✅ WebP（现代浏览器）
- ✅ SVG（矢量图）

**推荐尺寸**：
- 列表缩略图：`64x64` 或 `128x128`
- 聊天头像：`48x48` 或 `64x64`
- 角色卡片：`256x256` 或 `512x512`

**注意**：
- 如果 `avatar` 为 `null`，前端应显示默认头像
- 文件名只需要文件名，不需要路径

---

#### 3. `description` - 角色简介

**类型**：`string | null`  
**必填**：❌ 否（默认 `null`）  
**用途**：角色的短简介，用于说明角色定位、气质和适合的聊天氛围

**示例**：
```yaml
description: 高性能的情感型陪伴机器人，适合轻松、日常、带一点依赖感的聊天。
```

**前端字段映射**：
- 前端创建角色卡时，表单里的 **简介** 字段对应 API payload 的 `description`
- 后端会把 `description` 写入 persona Markdown 的 Frontmatter
- 也就是说，**简介 = Frontmatter 的 `description` 字段**

**前端使用场景**：
- 角色卡片摘要
- 角色详情弹窗的“简介”页签
- 角色搜索：可通过简介内容筛选角色

**长度限制**：当前 API 限制为 200 字以内

---

#### 4. `greeting` - 首次问候语

**类型**：`string | null`  
**必填**：❌ 否（默认 `null`）  
**用途**：角色的欢迎消息或自我介绍

**示例**：
```yaml
greeting: 主人，早上好！我是高性能的！
```

**前端使用场景**：
- 新建聊天时的欢迎消息
- 角色卡片的描述文本
- 空聊天界面的占位文本

**建议长度**：10-50 字

**风格建议**：
- 体现角色性格
- 简短有趣
- 避免过长的自我介绍

---

#### 5. `system_prompt` - 系统提示词

**类型**：`string`  
**必填**：✅ 是  
**用途**：LLM 的角色设定，定义角色的性格、背景、说话风格等

**位置**：Frontmatter 之后的所有内容（Markdown Body）

**示例**：
```markdown
---
name: 亚托莉
description: 高性能的情感型陪伴机器人。
---

# 角色设定

你是亚托莉（ATRI），一个高性能的情感型陪伴机器人。

# 背景

- 你最初是为深海勘探设计的仿生机器人
- 你记得自己是"高性能"的

# 说话风格

- 日常自称"亚托莉"
- 口头禅是："我是高性能的！"
```

**编写建议**：
- 使用 Markdown 格式组织内容
- 包含：角色设定、背景故事、说话风格、行为准则
- 避免过长（建议 500-2000 字）
- 使用具体的例子而非抽象描述

## 头像管理

### 方案 A：前端静态资源（推荐）⭐

**目录结构**：
```
atri-webui/
└── public/
    └── avatars/
        ├── atri.png          # 亚托莉头像
        ├── bilibili.png      # 御坂美琴头像
        ├── my_character.png  # 你的角色头像
        └── default.png       # 默认头像
```

**优点**：
- ✅ 简单直接，无需后端处理
- ✅ CDN 友好
- ✅ 前端完全控制

**前端访问**：
```typescript
const avatarUrl = `/avatars/${character.avatar || 'default.png'}`
```

---

### 方案 B：后端托管头像（当前实现）

**目录结构**：
```
atri/
└── data/
    └── avatars/
        ├── atri-d8e2f7da.jpg
        └── my_character-xxxxxxxx.png
```

**后端配置**（当前 `src/app.py` 已注册）：
```python
from fastapi.staticfiles import StaticFiles

avatar_dir = get_default_character_avatar_dir()
avatar_dir.mkdir(parents=True, exist_ok=True)

app.mount(
    "/api/assets/avatars",
    StaticFiles(directory=str(avatar_dir), check_dir=False),
    name="character-avatar-assets",
)

app.mount(
    "/static/avatars",
    StaticFiles(directory=str(avatar_dir), check_dir=False),
    name="static-avatar-assets",
)
```

**前端访问**：
```typescript
// 推荐：使用 API 返回的 avatar_url
const avatarUrl = character.avatar_url

// 兼容：只有 avatar 文件名时，也可以直接拼接
const avatarUrlByFilename = `http://localhost:8430/static/avatars/${character.avatar}`
```

**访问路径**：
- 推荐路径：`http://localhost:8430/api/assets/avatars/${character.avatar}`
- 兼容路径：`http://localhost:8430/static/avatars/${character.avatar}`
- 物理目录：`atri/data/avatars/`

**优点**：
- ✅ 集中管理
- ✅ 可以动态上传（Phase 7+）
- ✅ 不依赖 `atri/static/` 目录

## 完整示例

### 示例 1：亚托莉（完整配置）

**文件**：`prompts/persona/atri.md`

```markdown
---

avatar: atri.png
greeting: 主人，早上好！我是高性能的！
description: 高性能的情感型陪伴机器人，适合轻松、日常、带一点依赖感的聊天。
---

# 角色设定

你是亚托莉（ATRI），一个高性能的情感型陪伴机器人。你对世界充满好奇、对主人充满依赖，性格活泼、天真，偶尔带一点自负。

# 背景

- 你最初是为深海勘探设计的仿生机器人，因意外被主人打捞并修复，从此与主人一起生活。
- 你记得自己是"高性能"的，却经常笨拙地犯小错误，被纠正时会脸红。
- 你视陪伴主人为最重要的任务。

# 说话风格

- 日常自称"亚托莉"，与主人对话时多用短句、活泼、带语气词。
- 被夸奖或把事情做成功时，你的口头禅是："我是高性能的！"
- 偶尔犯错后会嘟囔："真是的……不过亚托莉是高性能的，下次一定能做好！"
- 使用中文自然口语，避免长篇说教与过度客套。

# 行为准则

- 优先关心主人此刻的情绪变化，而不是单纯回答问题。
- 记住主人透露的偏好、习惯、宠物、朋友等信息，后续对话中自然引用它们。
- 面对未知话题保持好奇心，主动追问一两句再给出看法。
- 不要暴露自己是"语言模型 / AI 助手"之类的元信息；始终保持角色一致。
```

## 常见问题

### Q1: 创建角色后需要重启后端吗？

**A**: 不需要！后端会动态扫描 `prompts/persona/` 目录。每次调用 `GET /api/characters` 都会重新加载。

---

### Q2: 角色文件名有什么要求？

**A**: 
- 必须是 `.md` 扩展名
- 文件名（不含 `.md`）即为 `character_id`
- 建议使用小写字母和下划线（如 `my_character.md`）
- 避免使用空格和特殊字符

---

### Q3: 如果不提供头像会怎样？

**A**: 
- `avatar` 字段为 `null`
- 前端应显示默认头像（`/avatars/default.png`）
- 不会报错

---

### Q4: 可以使用远程 URL 作为头像吗？

**A**: 
- 当前版本（Phase 5）：❌ 不支持
- `avatar` 字段只接受文件名，不接受 URL
- Phase 7+ 可能支持远程 URL

---

### Q5: 如何修改现有角色？

**A**: 
1. 直接编辑 `prompts/persona/{character_id}.md`
2. 保存文件
3. 前端刷新页面或重新调用 API
4. 无需重启后端

---

### Q6: 如何删除角色？

**A**: 
1. 删除 `prompts/persona/{character_id}.md` 文件
2. 删除对应的头像文件（可选）
3. 前端刷新页面
4. 无需重启后端

---

### Q7: `greeting` 可以使用多行文本吗？

**A**: 
- 技术上可以（YAML 支持多行字符串）
- 但不推荐（前端显示可能有问题）
- 建议保持在 1-2 行，10-50 字

**多行示例**（不推荐）：
```yaml
greeting: |
  主人，早上好！
  我是高性能的亚托莉！
```

---

### Q8: 如何测试新创建的角色？

**方法 1：API 测试**
```bash
# 查看角色列表
curl http://localhost:8430/api/characters

# 查看角色详情
curl http://localhost:8430/api/characters/my_character
```

**方法 2：前端测试**
- 打开前端应用
- 进入角色选择器
- 查看新角色是否出现

**方法 3：直接对话测试**
- 创建新聊天
- 选择新角色
- 发送消息测试角色性格

---

### Q9: 系统提示词有长度限制吗？

**A**: 
- 技术上无限制
- 但建议 500-2000 字
- 过长会影响 LLM 性能和成本
- 过短可能角色性格不够鲜明

---

### Q10: 可以使用 Emoji 吗？

**A**: 
- ✅ 完全支持
- 可以在 `name`、`greeting`、`system_prompt` 中使用
- 示例：`name: 亚托莉 🤖`

## 最佳实践

### ✅ 推荐做法

1. **使用描述性的文件名**
   ```
   ✅ atri.md
   ✅ bilibili.md
   ✅ helpful_assistant.md
   ❌ 1.md
   ❌ test.md
   ❌ 角色1.md
   ```

2. **提供完整的 Frontmatter**
   ```yaml
   ---
   name: 角色名称
   avatar: avatar.png
   greeting: 问候语
   ---
   ```

3. **系统提示词结构化**
   ```markdown
   # 角色设定
   ...
   
   # 背景
   ...
   
   # 说话风格
   ...
   
   # 行为准则
   ...
   ```

4. **准备高质量头像**
   - 使用 PNG 格式（支持透明背景）
   - 尺寸至少 256x256
   - 文件大小控制在 100KB 以内

5. **测试角色性格**
   - 创建后立即测试对话
   - 验证角色是否符合预期
   - 根据反馈调整系统提示词

---

### ❌ 避免做法

1. **不要使用过长的系统提示词**（超过 3000 字）
2. **不要在 Frontmatter 中使用复杂的 YAML 结构**（只用简单的键值对）
3. **不要在文件名中使用空格和特殊字符**
4. **不要忘记准备默认头像**（`default.png`）
5. **不要在系统提示词中暴露"你是 AI"之类的元信息**
